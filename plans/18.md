## Goal / scope

The pure, transport- and DB-agnostic foundation for world persistence: a sparse per-chunk
**edit-delta** codec that diffs a chunk's current voxels against its freshly regenerated base,
plus the world⇆bytes bridge that reconstructs a world from a seed + stored deltas. Implements the
storage *shape* ADR-0001 designed but deferred. **No DB, no DOM, no runtime behavior change** —
the game still reads the literal seed. Verified purely through unit tests.

**Out of scope (owned elsewhere):** #19 Prisma models / tRPC / base64 DB boundary · #20 auth-aware
load, persist queue, `GameStore.onCommit` wiring · #21 pause menu UI. This issue is pure logic,
unit-testable in Node.

## Corrections baked in (do not repeat the earlier draft's mistakes)

1. **No `Buffer`, no base64 in `src/game/**`.** The codec and bridge operate on `Uint8Array`
   only. `src/game/**` is imported by the browser r3f scene; `Buffer` is Node-only and would
   break the client build. Base64 ⇄ bytes conversion is a **#19 server-boundary** concern
   (`Buffer.from(data,'base64')` lives in the tRPC router). Keep this layer bytes-in / bytes-out.
2. **`World.chunks` is `private`.** Never write `world.chunks.get(...)`. Use `parseChunkKey(key)`
   then `world.getChunk(cx, cy, cz)` (returns `Chunk | undefined`).
3. **`generateWorld(config)` returns `GeneratedWorld` = `{ world, trees, size, spawn }`.** You must
   destructure `.world`. It is `generateWorld({ seed })` → `.world`, not a `World` directly.
4. **All-air chunks are never created by worldgen** (chunks are lazy; only content-bearing keys
   exist). `baseChunkBytes` must return a **zero-filled `CHUNK_VOLUME` array** when
   `getChunk` is `undefined`, not throw. Air === 0, so a zero array is the correct base.

## Deliverables

### 1. Chunk byte accessors — `src/game/chunk.ts`
- `snapshot(): Uint8Array` → `return this.blocks.slice()` (defensive copy).
- `load(bytes: Uint8Array): void` → if `bytes.length !== CHUNK_VOLUME` throw a clear
  `RangeError`; else `this.blocks.set(bytes)`.
- Keep `blocks` private. Export `CHUNK_VOLUME` from `chunk.ts` (the codec/bridge and tests need it).

### 2. Pure delta codec — `src/game/persistence/chunk-delta.ts` (new; Uint8Array only)
Byte format: `[uint8 version][uint16 count]( uint16 idx, uint8 val )*` → k edits = `3 + 3k` bytes.
Use `DataView` on an `ArrayBuffer` (browser-safe). Big-endian for the uint16s (pick one and be consistent).
- `export const CHUNK_DELTA_VERSION = 1`.
- `encodeChunkDelta(base: Uint8Array, current: Uint8Array): Uint8Array` — validate both are
  `CHUNK_VOLUME`; emit only indices where `base[i] !== current[i]`, value = `current[i]`.
- `decodeChunkDelta(base: Uint8Array, delta: Uint8Array): Uint8Array` — start from `base.slice()`;
  read version (reject if `!== CHUNK_DELTA_VERSION`); read count; replay `(idx,val)`; reject
  `idx >= CHUNK_VOLUME`. Do not mutate inputs.
- `isEmptyChunkDelta(delta: Uint8Array): boolean` — true when decoded count === 0
  (read the uint16 count field; do not guess from `.length`).

### 3. World⇆bytes bridge — `src/game/persistence/world-delta.ts` (new; Uint8Array only)
```
export interface ChunkDeltaRecord { chunkKey: ChunkKey; data: Uint8Array }
```
- `baseChunkBytes(seed: string, key: ChunkKey): Uint8Array` — `const { world } = generateWorld({ seed })`;
  `const { cx, cy, cz } = parseChunkKey(key)`; `const chunk = world.getChunk(cx, cy, cz)`;
  return `chunk ? chunk.snapshot() : new Uint8Array(CHUNK_VOLUME)` (zero = all Air).
- `computeChunkDelta(seed, key, current: Uint8Array): Uint8Array` →
  `encodeChunkDelta(baseChunkBytes(seed, key), current)`.
- `hydrateWorld(seed, deltas: ChunkDeltaRecord[]): World` — `const { world } = generateWorld({ seed })`;
  `applyStoredDeltas(world, seed, deltas)`; return `world`.
- `applyStoredDeltas(world: World, seed: string, deltas: ChunkDeltaRecord[]): void` — for each
  delta: `base = baseChunkBytes(seed, key)`; `bytes = decodeChunkDelta(base, data)`; obtain/create
  the chunk in `world` for that key and `chunk.load(bytes)`. Decode against the **seeded base**,
  not the chunk's current bytes, so it is order-independent and idempotent. (If `World` has no
  public "get-or-create chunk" method, add a minimal one or use `setBlock` per changed voxel — do
  not reach into private `chunks`.)

### 4. Determinism golden test — extend `src/game/__tests__/worldgen.test.ts`
- Pin a hash (e.g. `node:crypto` `createHash('sha256')` — test file is Node, so `crypto`/`Buffer`
  are fine **in tests**) or byte-snapshot of `generateWorld({ seed:"<fixed>" }).world` for a couple
  of chunks (spawn chunk + an edge chunk). Compute the real value on first run and pin it.
- Comment loudly: **drift here means every stored delta diffs against the wrong base → silent save
  corruption. Do NOT silence; treat a needed update as a red flag and investigate root cause.**

### 5. Tests — test → break-the-code-to-confirm-failure → fix, for every module
- `src/game/__tests__/chunk.test.ts` (extend): mutating the `snapshot()` result doesn't touch the
  chunk; write→snapshot→load round-trip; `load` rejects wrong length (both `<` and `>`).
- `src/game/__tests__/persistence/chunk-delta.test.ts` (new): round-trip identity
  `decodeChunkDelta(base, encodeChunkDelta(base, current))` deep-equals `current` for seeded-random
  edited chunks; untouched → empty delta (count 0), decodes to base; single-edit compactness
  (length `3+3`, not 4096); determinism vs seed; out-of-range decode guard; version guard.
  **Break-the-code:** flip encode `!==`→`===`, confirm round-trip fails, revert.
- `src/game/__tests__/persistence/world-delta.test.ts` (new): hand-edit a world, compute deltas,
  `hydrateWorld` reconstructs it exactly; unedited chunk → empty delta; disjoint-chunk ordering
  independence (deltas applied in any order → same world).

## Acceptance criteria (from the issue — all must pass)
- [ ] `decode(base, encode(base, current))` deep-equals `current` for arbitrary edited chunks
- [ ] Untouched chunk → empty delta → decodes back to base
- [ ] Single-voxel edit → small sparse delta (not a full-grid blob)
- [ ] Applying stored deltas onto a regenerated world reconstructs a hand-edited world exactly
- [ ] `decode` rejects out-of-range voxel index; `Chunk.load` rejects wrong-length array
- [ ] Pinned golden-hash test fails if `generateWorld` output drifts for a fixed seed
- [ ] `pnpm test`, `pnpm check`, `pnpm build` green; **no runtime behavior change**
- [ ] Each new module followed test → break-to-confirm → fix

## Risks & mitigations
| Risk | Mitigation |
| --- | --- |
| Determinism drift silently corrupts every delta | Pinned golden-hash test; loud red-flag comment |
| `Buffer`/base64 leaking into client bundle | Codec/bridge are Uint8Array-only; base64 deferred to #19 |
| Compile break from private `World.chunks` | Use `getChunk`/`parseChunkKey`; add minimal accessor if needed |
| Codec off-by-one | Round-trip identity tests + decode index/version guards + break-the-code check |
| Non-idempotent apply order | Decode each delta against the seeded base, not current bytes |
