# Plan: #15 — Extract pure interaction module from BlockTargeting (eye↔feet convention + face-normal placement)

**Issue:** #15 Extract pure interaction module from BlockTargeting (eye↔feet convention + face-normal placement)
**Branch:** `feat/architecture-deepening` (implement on a topic branch off it, e.g. `feat/15-extract-interaction`)
**Type:** Pure extraction / consolidation. **Zero gameplay change.**

## Objective & non-goals

**Objective.** The calling logic that turns a raycast into a `Command` currently lives inside the untested `BlockTargeting` React component (`src/game/render/block-target.tsx`). Extract it into a pure, plain-Vitest-testable interaction module, and give the eye↔feet `EYE_HEIGHT` convention exactly one home shared by both the interaction module and the player controller. Specifically move out of the component:

1. The **eye→feet** conversion (`eye.y - EYE_HEIGHT`) — the inverse of what `player-controller.tsx` does (`feet.y + EYE_HEIGHT`).
2. The **face-normal → place-cell snap** (`target + Math.round(normal)`), which encodes the translation-only chunk-transform assumption.
3. **Target / place-cell validity** (reach gate + "no face normal ⇒ no place cell").
4. The **empty-slot inventory gate** (right-click no-ops when the selected slot is empty).
5. Assembly of the exact `Command` + `from` + `reach` + `playerPosition` handed to `store.apply`.

**Non-goals.**
- No behavior change: identical reach gating, break target, place cell, eye→feet math, empty-slot rejection, and identical `from`/`reach`/`playerPosition` passed to `store.apply`.
- The three.js raycast, `instanceId`→coord resolution (`readInstanceCoord`), pointer-lock/pointer/keyboard/wheel event wiring, and the wireframe outline stay in `BlockTargeting`.
- No new `Command` variants, no `applyCommand`/`canBreak`/`canPlace`/`WorldStore.apply` signature changes. `#16`'s picking contract (`instance-picking.ts`) is untouched.
- No change to the inventory model, hotbar selection, or `crosshair-overlay`.

All symbol names below are grep-verified against the post-#13/#14/#16 tree on `feat/architecture-deepening`. The only *new* names are the interaction module's own types/functions and one relocated constant.

## Target design

### Decision 1 — `EYE_HEIGHT`'s single home: `src/game/player/player-box.ts`

**Move `export const EYE_HEIGHT = 1.6;` from `step-player.ts` into `player-box.ts`.** Rationale:

- `player-box.ts` is the dependency-light player-body-geometry leaf that **both** consumers already transitively import: `player-controller.tsx` (via `step-player`) and the new `interaction.ts` (directly). Making it the home means the eye↔feet convention has one literal declaration used symmetrically by the controller (feet→eye) and the interaction module (eye→feet), satisfying the acceptance criterion.
- It imports only `aabb` types, so no import cycle is introduced (`interaction.ts` → `player-box`; `command.ts` → `player-box` already exists; `player-box` → `aabb` only).
- **Naming note (call-out, not a blocker):** #13 framed `player-box.ts` as *collision-box* knowledge and kept `EYE_HEIGHT` out on that basis. #15's issue explicitly overrides that and suggests this module. Treat `player-box.ts` post-#15 as "player body geometry" (collision box **and** eye offset). *Alternative considered:* keep `EYE_HEIGHT` in `step-player.ts` and re-export, or a new `player-eye.ts` — rejected because a re-export leaves two apparent homes (dilutes "exactly one home"), and a new module is heavier than the issue's own suggestion. **Chosen: player-box.ts.**

### Decision 2 — new pure module `src/game/interaction.ts`

Pure, no three.js / React / DOM. Accepts plain `{x,y,z}` data; the component maps `THREE.Intersection` → the plain `RaycastHit` at the boundary.

```ts
import {
  DEFAULT_REACH,
  type Command,
  type Vec3,
} from "~/game/command";
import type { Inventory } from "~/game/inventory";
import { EYE_HEIGHT } from "~/game/player/player-box";

/** Plain-data camera pose. Only the eye position is needed today: this module
 *  does no direction math — the raycast that produced `hit` already encoded
 *  look direction. Orientation is intentionally omitted (the issue's "as
 *  needed"); add a field here if a future rule needs the look angle. */
export interface CameraPose {
  readonly eye: Vec3;
}

/** Plain-data raycast hit — the component maps THREE.Intersection → this. */
export interface RaycastHit {
  /** World-space integer block coord of the hit cell (InstanceCoord.world). */
  readonly cell: Vec3;
  /** Object-space face normal of the hit face. Equals the world-space normal
   *  here because the chunk transform chain is translation-only (documented
   *  in block-target.tsx's header); a translation never rotates a direction.
   *  Optional: a hit may carry no usable face normal. */
  readonly faceNormal?: Vec3;
  /** Ray length from camera to hit, world units — the reach-gate input. */
  readonly distance: number;
}

export type InteractionAction = "break" | "place";

/** The two cells a hit resolves to, reach-gated. Both null when the hit is
 *  out of reach; `place` is additionally null when the hit carried no face
 *  normal. Used by the per-frame outline and internally by deriveInteraction. */
export interface TargetCells {
  readonly target: Vec3 | null;
  readonly place: Vec3 | null;
}

/** A ready-to-apply command plus the exact extra args WorldStore.apply needs. */
export interface InteractionCommand {
  readonly command: Command;
  readonly from: Vec3;
  readonly reach: number;
  readonly playerPosition: Vec3;
}

/** Reach gate + face-normal→place-cell snap. Encodes the translation-only
 *  assumption via Math.round (object-space normal == world-space normal here). */
export function resolveTargetCells(
  hit: RaycastHit | null,
  reach: number = DEFAULT_REACH,
): TargetCells {
  if (!hit || hit.distance > reach) {
    return { target: null, place: null };
  }
  const target = hit.cell;
  const n = hit.faceNormal;
  const place = n
    ? {
        x: target.x + Math.round(n.x),
        y: target.y + Math.round(n.y),
        z: target.z + Math.round(n.z),
      }
    : null;
  return { target, place };
}

/** Eye↔feet conversion + target/place validity + empty-slot inventory gate +
 *  Command/from/reach/playerPosition assembly. Returns null when there's
 *  nothing valid to do (out of reach, no face for a place, empty slot). */
export function deriveInteraction(
  pose: CameraPose,
  hit: RaycastHit | null,
  inventory: Inventory,
  action: InteractionAction,
  reach: number = DEFAULT_REACH,
): InteractionCommand | null {
  const { target, place } = resolveTargetCells(hit, reach);
  const eye = pose.eye;

  if (action === "break") {
    if (!target) return null;
    // from = eye; playerPosition = eye (never read by BreakBlock, matches the
    // defaulted 2-arg call today); reach = DEFAULT_REACH (matches default).
    return {
      command: { type: "BreakBlock", at: target },
      from: eye,
      reach,
      playerPosition: eye,
    };
  }

  // action === "place"
  if (!place) return null;
  const slot = inventory.slots[inventory.selected];
  if (!slot?.block) return null; // empty-slot gate
  // eye→feet: inverse of player-controller's feet→eye (single EYE_HEIGHT home).
  const playerPosition: Vec3 = { x: eye.x, y: eye.y - EYE_HEIGHT, z: eye.z };
  return {
    command: { type: "PlaceBlock", at: place, block: slot.block },
    from: eye,
    reach,
    playerPosition,
  };
}
```

**Behavior-equivalence to today (line-by-line):**
- Reach gate: `hit.distance > reach ⇒ null` ≡ `hit.distance <= DEFAULT_REACH` gate at `block-target.tsx:121`.
- Break: `store.apply(cmd, eye)` (2-arg) ≡ `store.apply(cmd, eye, DEFAULT_REACH, eye)` (4-arg) because `BreakBlock` ignores `playerPosition` and `reach` default is `DEFAULT_REACH`.
- Place: `at = target + round(normal)`, `from = eye`, `reach = DEFAULT_REACH`, `playerPosition = eye - EYE_HEIGHT` — identical to `block-target.tsx:191-201`.
- Empty-slot: `!slot?.block ⇒ null` ≡ `block-target.tsx:184`.
- No-face-normal place: `place === null ⇒ null` ≡ `placeAtRef === null ⇒ return` at `block-target.tsx:181`.

### Decision 3 — thinned `BlockTargeting`

Keeps: candidate gather + `raycaster.setFromCamera(NDC_CENTER, camera)` + `intersectObjects`; `readInstanceCoord` resolution; pointer-lock guards; `pointerdown`/`contextmenu`/`keydown`/`wheel` wiring; hotbar `selectSlot`/`cycleSelection`; the `EdgesGeometry`/`LineSegments` outline. Removes all derivation math.

```ts
// imports: drop EYE_HEIGHT; add interaction module + keep DEFAULT_REACH/Vec3.
import { DEFAULT_REACH } from "~/game/command";
import {
  deriveInteraction,
  resolveTargetCells,
  type RaycastHit,
} from "~/game/interaction";

// One ref replaces targetRef + placeAtRef: the last frame's plain hit.
const hitRef = useRef<RaycastHit | null>(null);

useFrame(() => {
  // ...gather candidates + raycast (unchanged)...
  const hit = hits[0];

  let mapped: RaycastHit | null = null;
  if (hit && hit.instanceId !== undefined) {
    const coord = readInstanceCoord(hit.object as THREE.InstancedMesh, hit.instanceId);
    if (coord) {
      const n = hit.face?.normal;
      mapped = {
        cell: coord.world,
        faceNormal: n ? { x: n.x, y: n.y, z: n.z } : undefined,
        distance: hit.distance,
      };
    }
  }
  hitRef.current = mapped;

  const { target } = resolveTargetCells(mapped, DEFAULT_REACH); // outline only
  const outline = outlineRef.current;
  if (!outline) return;
  if (target) {
    outline.visible = true;
    outline.position.set(target.x + 0.5, target.y + 0.5, target.z + 0.5);
  } else {
    outline.visible = false;
  }
});

// pointerdown handler:
const handlePointerDown = (event: PointerEvent) => {
  if (document.pointerLockElement !== canvas) return;
  const action = event.button === 0 ? "break" : event.button === 2 ? "place" : null;
  if (!action) return;
  const pose = { eye: { x: camera.position.x, y: camera.position.y, z: camera.position.z } };
  const inventory = store.getInventorySnapshot();
  const result = deriveInteraction(pose, hitRef.current, inventory, action, DEFAULT_REACH);
  if (!result) return;
  store.apply(result.command, result.from, result.reach, result.playerPosition);
};
```

Notes:
- Mapping `THREE.Intersection` → `RaycastHit` (world coord via `readInstanceCoord`, plain-copied `face.normal`, `distance`) is the only boundary; it is not derivation math.
- `hitRef` holds the last frame's hit; the click re-resolves it deterministically inside `deriveInteraction` — identical outcome to today's `targetRef`/`placeAtRef` (both were last-frame values).
- `contextmenu`/`keydown`/`wheel`/hotbar handlers and the `EdgesGeometry` memo are unchanged.

## New test spec — `src/game/__tests__/interaction.test.ts` (plain Vitest, no DOM/renderer)

Imports: `deriveInteraction`, `resolveTargetCells`, types from `~/game/interaction`; `BlockType` from `~/game/blocks`; `createInventory`, `addDrop` from `~/game/inventory`; `DEFAULT_REACH` from `~/game/command`; `EYE_HEIGHT` from `~/game/player/player-box`.

Helpers: `const EYE = { x: 0.5, y: 10, z: 0.5 };` `pose = { eye: EYE }`; `const CELL = { x: 0, y: 8, z: 0 };` (within `DEFAULT_REACH` of EYE); an inventory with a block in the selected slot via `addDrop(createInventory(), BlockType.Dirt)`.

- **`resolveTargetCells` — reach gate:** `distance = DEFAULT_REACH + 0.1` ⇒ `{ target: null, place: null }`; `distance = DEFAULT_REACH` ⇒ `target === cell` (boundary inclusive).
- **`resolveTargetCells` — normal snap + rounding:** `faceNormal = { x: 0.0001, y: 0.9999, z: 0 }` ⇒ `place = { x: cell.x, y: cell.y + 1, z: cell.z }` (proves the `Math.round` translation-only snap). A `-x` normal `{ -1, 0, 0 }` ⇒ `place.x === cell.x - 1`.
- **`resolveTargetCells` — no face normal:** `faceNormal` omitted ⇒ `target === cell` but `place === null`.
- **Break derivation:** in-reach hit, `action "break"` ⇒ `command = { type: "BreakBlock", at: cell }`, `from === EYE`, `playerPosition === EYE`, `reach === DEFAULT_REACH`.
- **Place derivation via face normal:** in-reach hit with `faceNormal {0,1,0}`, non-empty inventory, `action "place"` ⇒ `command = { type: "PlaceBlock", at: {cell.x, cell.y+1, cell.z}, block: <selected> }`, `from === EYE`, `reach === DEFAULT_REACH`.
- **Reach/validity nulls:** out-of-reach hit ⇒ `deriveInteraction` returns `null` for both `"break"` and `"place"`; `hit = null` ⇒ `null`; place with no face normal ⇒ `null`.
- **Empty-slot place rejection:** valid in-reach hit + face normal, but `inventory = createInventory()` (selected slot empty) + `action "place"` ⇒ `null`. (Break still succeeds with the same empty inventory — proves the gate is place-only.)
- **eye↔feet round-trip:** pick a feet position `F`; set `eye = { x: F.x, y: F.y + EYE_HEIGHT, z: F.z }`; `deriveInteraction(place)` ⇒ `result.playerPosition` deep-equals `F`. This ties both directions to the single `EYE_HEIGHT` home (import from `player-box`).
