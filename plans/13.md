## Target design (the one home for each fact)

### Fact 2 → new module `src/game/player/player-box.ts`

One home for collision-box dimensions **and** the feet-position constructor; a change to player size touches only this file.

```ts
// src/game/player/player-box.ts
import type { Box3, Vec3 } from "~/game/player/aabb";

/** Player collision box: 0.6 wide x 1.8 tall x 0.6 deep. */
export const PLAYER_WIDTH = 0.6;
export const PLAYER_HEIGHT = 1.8;
export const PLAYER_DEPTH = 0.6;

/** Build the player's world-space AABB from its feet-center position. */
export function boxFromFeetPosition(
  position: Vec3,
  width: number,
  height: number,
  depth: number,
): Box3 {
  const halfW = width / 2;
  const halfD = depth / 2;
  return {
    min: { x: position.x - halfW, y: position.y, z: position.z - halfD },
    max: { x: position.x + halfW, y: position.y + height, z: position.z + halfD },
  };
}
```

Function body is copied **verbatim** from `aabb.ts:43-55`. `player-box.ts` imports the geometry types `Box3`/`Vec3` from `aabb.ts` (no cycle: aabb does not import player-box).

- `aabb.ts`: delete `boxFromFeetPosition` (lines 42-55) and its doc comment. `Box3`, `Vec3`, `translateBox`, `boxIntersectsSolid`, `boxesOverlap`, `sweepAxis`, `SweepResult` stay.
- `step-player.ts`: remove the three dim declarations (`:62-64`); import `boxFromFeetPosition`, `PLAYER_WIDTH`, `PLAYER_HEIGHT`, `PLAYER_DEPTH` from `~/game/player/player-box`. (`EYE_HEIGHT` declaration `:66` stays in step-player.) `boxFromFeetPosition` also drops out of the aabb import block at `:17-24`.
- `command.ts`: import `PLAYER_WIDTH/HEIGHT/DEPTH` **and** `boxFromFeetPosition` from `~/game/player/player-box` (replacing the aabb import of `boxFromFeetPosition` and the step-player import of the dims). Satisfies AC "placement validation no longer imports dimension constants from the physics stepper."

### Fact 3 → declare in `src/game/coords.ts`, re-export from `src/game/chunk.ts`

**One declaration lives in `coords.ts`** (`export const CHUNK_SIZE = 16;`, unchanged). Rationale: `coords.ts` is the zero-dependency spatial-math leaf; `CHUNK_SIZE` is a coordinate-space parameter it uses in every conversion. Keeping the declaration there preserves `coords.ts`'s purity. If `chunk.ts` were canonical instead, `coords.ts` would have to import `chunk.ts` (→ `blocks.ts`), transitively polluting the leaf. `chunk.ts` already sits above `coords` conceptually and re-exports cleanly.

- `chunk.ts`: delete `export const CHUNK_SIZE = 16;` (`:11`); add `import { CHUNK_SIZE } from "~/game/coords";` (for internal `CHUNK_VOLUME`/`localIndex`/`inBounds` use) and `export { CHUNK_SIZE };` (re-export). No cycle: `coords` imports nothing.

**All existing importer paths keep working unchanged** — importers from `~/game/chunk` still resolve `CHUNK_SIZE` via the re-export; importers from `~/game/coords` are unaffected. Zero import-site edits, zero test edits for this fact.

## Rollback / risk notes

- **What could accidentally change behavior:** (a) mistyping a copied constant or the box-constructor math during the move; (b) leaving a stale duplicate declaration so an importer silently binds the wrong `16`/dim (masked drift); (c) introducing an import cycle. Mitigations: constants/function are moved verbatim (diff-reviewable); `grep` checks in the checklist prove single declarations; dependency direction is strictly `command/step-player/aabb → voxel`, `player-box → aabb`, `chunk → coords` — all pointing at leaves, no cycle (`coords` and `voxel` import nothing but `blocks`).
- **Type-only churn is runtime-inert:** the `VoxelWorld → VoxelReader` rename and all annotation changes are erased at build; they cannot alter behavior, only compilation.
- **Rollback:** the change is confined to 7 source files + 2 test files + 2 new files. Revert the topic branch; no data, schema, or wire-contract (`command-schema.ts`) is touched, so rollback is a plain `git revert`/branch-drop with no migration.
