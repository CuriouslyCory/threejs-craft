## The wiring challenge (important)
The pause menu lives in `LockOverlay` (inside `WorldSession`, the INNER component), but Save/New
Game actions live in the OUTER `GameScene` (which owns `queue`, `setHydrated`, the `newGame`
mutation, and the signed-in/out flag). So this issue must **thread new props from outer → inner →
overlay**: add `onSave?`, `onNewGame?`, `saveStatus`, and an `isSignedIn`/`isEphemeral` flag to
`WorldSessionProps` and forward them into `<LockOverlay>`. Keep `WorldSession`'s existing per-mount
behavior intact. Do not move `lockState` out of the inner component.

## Deliverables

### 1. Pure menu reducer — `src/game/render/lock-overlay-menu.ts` (new, no DOM)
Extract the confirm/save state machine so it unit-tests without DOM:
- States: `idle` → (New Game clicked) `confirming` → (`Start fresh`) `working` → `idle`; `confirming`
  → (`Never mind`/cancel) `idle`. Plus `saveStatus`: `none` → `saving` → `saved` | `error`.
- Export a reducer `(state, action) => state` + action creators (or a small pure API). Keep it total
  and side-effect-free; the component dispatches and runs the async effects.
- Tests `src/game/__tests__/render/lock-overlay-menu.test.ts`: idle→confirming→working→idle happy
  path; cancel path; saveStatus transitions (saving→saved, saving→error, and a later save clears
  error). Break-the-code: drop the `confirming` guard so New Game fires immediately → confirm the
  "requires confirm" test fails → revert.

### 2. Pause menu UI — `src/game/render/lock-overlay.tsx` `paused` branch
Replace the resume text with a vertical, cat-themed button menu:
- Container `role="dialog"` `aria-modal="true"` labelled by the "Paused" heading. Backdrop keeps
  `onClick={onRequestLock}` (empty-space click = resume); **every inner control `stopPropagation`s**
  so a button click doesn't bubble to the resume backdrop.
- **Resume** — `onClick={onRequestLock}` from the button itself (satisfies the Pointer Lock
  user-gesture requirement — must be the button's own gesture, not a programmatic call).
- **Save** — signed-in: `onClick={onSave}`, shows `saveStatus` (`Saving…` / `World saved 🐾` /
  `Couldn't save…`). Signed-out: relabel **"Sign in to save"** as a link to
  `/api/auth/signin?callbackUrl=/game`, with an `aria-describedby` caption "Your world is saved to
  your account."
- **New Game** — two-step confirm ("Start a new world? This grows a fresh world from a new seed and
  replaces your current one." → `Start fresh` / `Never mind`, with **cancel default-focused**).
  Disabled while the mutation is in flight (double-tap guard).
- **Settings** — **inert but accessible: `aria-disabled="true"` (NOT native `disabled`)** so it stays
  focusable and screen readers announce "Settings — coming soon"; visually dimmed
  `opacity-50 cursor-not-allowed`; its handler is a no-op.
- Focus management: on entering `paused`, focus moves to **Resume**; Tab is trapped cycling
  Resume → Save → New Game → Settings (and back). Escape from the menu resumes (or is a no-op if the
  browser already maps Esc to exit pointer lock — verify current behavior).
- New `LockOverlayProps`: `onSave?`, `saveStatus?`, `onNewGame?`, `isNewGameInFlight?`, `isSignedIn`
  (or `isEphemeral`). Keep `paused` a pure function of props (no data fetching in the overlay).

### 3. Wire actions — `src/app/game/game-scene.tsx`
- Outer `GameScene`:
  - `onSave = async () => { dispatch(saving); try { await queue.flush(); await utils.world.status.fetch(); dispatch(saved) } catch (e) { dispatch(error) } }`.
  - `onNewGame = async () => { const {worldId, seed} = await newGame.mutateAsync(); await utils.world.load.invalidate(); setHydrated({worldId, seed, deltas: EMPTY_DELTAS}) }` → the `key` bump
    remounts `WorldSession` (fresh world/spawn/inventory, empty deltas). After remount the overlay
    returns to the "click to play" path so pointer lock is re-acquired by a fresh gesture.
  - **Signed-out New Game** = local ephemeral reseed: mint a client-side random seed (hold an
    ephemeral-seed in outer state, default `EPHEMERAL_SEED`), change it AND the `WorldSession` key so
    it remounts. No server call. Harmless + delightful. (The menu's saveStatus machine can live in
    the outer component or the overlay — put it wherever keeps the overlay a pure prop function;
    the reducer from Deliverable 1 is the source of truth.)
  - `beforeunload` best-effort `queue.flush()` (fire-and-forget; don't block unload).
  - Pass `onSave`/`onNewGame`/`saveStatus`/`isSignedIn`/`isNewGameInFlight` down through
    `WorldSession` to `LockOverlay`.
- Preserve zero-React-re-renders-per-frame: all of this is user-gesture driven, never `useFrame`.
